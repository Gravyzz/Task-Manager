# Task Manager CLI

Консольный менеджер задач на чистой Java. Позволяет создавать задачи с приоритетами, отмечать их выполненными, фильтровать и сортировать, а также сохранять между запусками в текстовый файл.

Проект написан без сторонних библиотек и фреймворков — только стандартная Java. Цель — продемонстрировать владение ядром языка: ООП, коллекциями, Stream API, обработкой исключений и работой с файлами.

---

## Возможности

- Добавление задач с названием и приоритетом (LOW / MEDIUM / HIGH)
- Просмотр всех задач
- Отметка задачи как выполненной
- Удаление задачи по ID
- Сортировка задач по приоритету (от высокого к низкому)
- Автоматическое сохранение и загрузка задач из файла — данные не теряются между запусками
- Обработка ошибок ввода: неверный приоритет, несуществующий ID, нечисловой ввод

## Технологии и концепции

| Область | Что использовано |
|---|---|
| ООП | Инкапсуляция, иммутабельные объекты, перегрузка конструкторов, `equals` / `hashCode` |
| Enum | `Priority` и `Status` с полями и методами |
| Коллекции | `List`, `ArrayList`, защитное копирование |
| Stream API | `filter`, `map`, `sorted`, `collect`, `mapToInt`, `Comparator` |
| Исключения | Собственные unchecked-исключения, обработка checked `IOException` |
| File I/O | `java.nio.file.Files` — сохранение и загрузка задач |
| Архитектура | Разделение на слои: модель / логика / хранение / интерфейс |

## Структура проекта

```
src/
├── model/
│   ├── Task.java                  # Модель задачи (иммутабельная, кроме статуса)
│   ├── Priority.java              # Enum приоритета (LOW, MEDIUM, HIGH)
│   └── Status.java                # Enum статуса (TODO, DONE)
│
├── service/
│   └── TaskManager.java           # Логика: добавление, удаление, поиск, фильтрация, сортировка
│
├── repository/
│   └── TaskRepository.java        # Сохранение и загрузка задач из файла
│
├── exception/
│   ├── InvalidTaskException.java  # Некорректные данные задачи
│   └── TaskNotFoundException.java # Задача с указанным ID не найдена
│
├── ui/
│   └── ConsoleUI.java             # Консольный интерфейс: меню и обработка ввода
│
└── Main.java                      # Точка входа
```

Каждый слой отвечает за свою задачу:
- **model** — данные (что такое задача)
- **service** — бизнес-логика (что с задачами можно делать)
- **repository** — хранение (как сохранить и загрузить)
- **ui** — взаимодействие с пользователем (как показать и спросить)

Такое разделение упрощает поддержку: например, заменить хранение в файле на базу данных можно, не затрагивая логику и интерфейс.

## Архитектурные решения

**Иммутабельность модели.** В классе `Task` поля `id`, `title`, `priority`, `createdAt` объявлены как `final` — они не меняются после создания. Изменяется только `status` (через метод `markDone()`), поэтому он единственный не `final`. Это осознанный выбор: неизменяемые объекты безопаснее и предсказуемее.

**Два конструктора задачи.** Один — для создания новых задач (статус автоматически `TODO`, время — текущее), второй — для восстановления задачи из файла со всеми сохранёнными полями. Основной конструктор делегирует полному через `this(...)`, чтобы не дублировать валидацию.

**Защитное копирование.** Метод `getAllTasks()` возвращает копию внутреннего списка, а не сам список. Это защищает данные менеджера от изменений извне в обход его методов.

**Транзакционность при добавлении.** Задача добавляется в список только после успешного создания с валидацией. Если данные некорректны — выбрасывается исключение, и состояние менеджера не меняется.

**Генерация ID после загрузки.** При загрузке задач из файла счётчик `nextId` устанавливается на максимальный существующий ID плюс один, чтобы новые задачи не конфликтовали со старыми.

## Запуск

Требуется Java 14 или выше (используется стрелочный `switch`).

Самый простой способ — открыть проект в IntelliJ IDEA и запустить класс `Main`.

При первом запуске файл `tasks.txt` создаётся автоматически после добавления первой задачи.

## Пример работы

```
Загружено задач: 2

=== Менеджер задач ===
1 - Добавить задачу
2 - Показать все задачи
3 - Отметить выполненной
4 - Удалить задачу
5 - Показать по приоритету
0 - Выход
Выберите действие: 1
Название задачи: Купить продукты
Приоритет (LOW/MEDIUM/HIGH): HIGH
Добавлена: [#3] Купить продукты | Приоритет: Высокий | Не выполнено

Выберите действие: 2
[#1] Помыть машину | Приоритет: Низкий | Выполнено
[#2] Позвонить врачу | Приоритет: Средний | Не выполнено
[#3] Купить продукты | Приоритет: Высокий | Не выполнено

Выберите действие: 0
Задачи сохранены
До свидания!
```

После выхода и повторного запуска все задачи загружаются из файла, включая их статусы.

## Формат хранения

Задачи сохраняются в `tasks.txt`, по одной на строку, поля разделены символом `;`:

```
id;название;приоритет;статус;время_создания
```

Пример:

```
1;Помыть машину;LOW;DONE;1782202008979
2;Позвонить врачу;MEDIUM;TODO;1782202008980
```
